# Design Notes

## Why CommonJS + ESM hybrid?

The project was generated by AI tooling at different points in time. `visibility/` was generated later with ESM syntax while the older modules use CommonJS. Eventually, the entire project should migrate to ESM. Until then:

- **DO NOT** add ESM files outside `visibility/`
- **DO NOT** add CommonJS files inside `visibility/`
- Use Webpack to bundle both systems together for browser delivery

## Why no React installed?

ESLint is configured with React plugins (`eslint-plugin-react`, `eslint-plugin-react-hooks`) but React is not in `package.json`. The `main-menu.js` is vanilla JS DOM manipulation. If you want to use React, install it and update the router.

## BSP Dungeon Generation Design

The BSP approach was chosen over cellular automata or random walk because:
1. Guarantees connected rooms (no unreachable areas)
2. Controllable room sizes via `minRoomSize`/`maxRoomSize`
3. Natural hierarchical structure for multi-level depth scaling

The `maxRatio` parameter in `BSPNode.split()` is currently ignored — it was intended to prevent excessively skinny partitions. It should be re-enabled when polishing.

## Visibility System

Uses Recursive Shadowcasting (Bjorn Bergstrom algorithm variant) rather than raycasting because:
- O(n) complexity vs O(n²) for raycasting
- Handles arbitrary obstacle shapes
- Produces smooth visibility polygons

The system is not yet integrated with the dungeon generator — tiles from `dungeon.js` need to be converted to `Segment` obstacles for `FogOfWar`.

## Known Technical Debt

1. **Module system split** — Unify to ESM
2. **No type system** — TypeScript is installed but unused. Add `tsconfig.json` and migrate gradually
3. **Skipped tests** — Need `src/database` and `src/app` implementations
4. **Dead code in FogOfWar.render()** — `visibleColor` was removed; rendering logic is incomplete
5. **ShadowCaster reserve calculations** — `p1`/`p2` boundary points computed but unused (kept for future quad-tree optimization)
6. **No game loop** — The roguelike core has generators but no runtime
7. **Corridor styles** — `corridorStyle` uses cryptic string values ('s', 'l', 'simple') instead of constants
