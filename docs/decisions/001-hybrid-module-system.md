# ADR-001: Hybrid Module System

## Status

Accepted (technical debt acknowledged)

## Date

2026-04-27 (retroactively documented)

## Context

The codebase contains two incompatible JavaScript module systems:
- `src/visibility/` — ES Modules (`import`/`export`)
- `src/` (all other files) — CommonJS (`require`/`module.exports`)

This occurred because the codebase was generated by AI tooling in multiple sessions:
1. Early sessions generated core modules (dungeon, bsp, room, etc.) in CommonJS
2. A later session generated the visibility system using ESM syntax

The project currently works because:
- Webpack bundles ESM and CJS together for browser delivery
- `visibility/` is not currently imported by any CJS module
- If integration were attempted, it would break at runtime in Node.js

## Decision

**Keep the hybrid system short-term, migrate entirely to ESM medium-term.**

Rationale:
- The immediate priority was stabilizing broken tests and tooling, not architectural refactors
- Webpack handles the mix for browser delivery
- Full ESM migration is planned in [module-migration.md](../module-migration.md)
- Until migration, enforce the boundary: no ESM outside `visibility/`, no CJS inside `visibility/`

## Consequences

**Positive:**
- Short-term velocity preserved — fixed 1200 lint errors without risking regressions from a full rewrite
- Webpack bundling works correctly
- `visibility/` is architecturally clean ESM for future migration

**Negative:**
- Agents must remember which directory uses which system
- Cannot share code between `visibility/` and rest of `src/` without a bundler
- ESLint config must handle both systems
- Confusing for new contributors / agents

## Re-evaluation

Re-evaluate after Phase 3 (Module Unification) of [roadmap.md](../roadmap.md). Target: migrate everything to ESM within 1-2 development days.

## Alternatives Considered

1. **Migrate visibility/ to CJS** — Rejected. CJS is legacy; moving toward ESM is forward-looking
2. **Big-bang ESM rewrite** — Rejected at this stage. No integration tests for dungeon+visibility, too risky
3. **Keep hybrid permanently** — Rejected. Ongoing maintenance complexity
